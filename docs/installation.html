When you first open Claude Owl, macOS may show this warning:

"Claude Owl cannot be opened because the developer cannot be verified."

This is normal for unsigned apps. Here's how to open it:

Option 1:

Right-click and Open

  1. 1. Right-click (or Control-click) the Claude Owl icon
  2. 2. Select "Open" from the menu
  3. 3. Click "Open" in the dialog that appears
Option 2:

System Settings

  1. 1. Open System Settings (or System Preferences)
  2. 2. Go to Privacy & Security
  3. 3. Scroll down to see the security warning
  4. 4. Click "Open Anyway"

üö® "App is damaged" error?

If you see "Claude Owl is damaged and can't be opened" or "You should move it to the Trash", this is a macOS security restriction on unsigned apps downloaded from the internet.

‚úÖ Verified Fix: Run this Terminal command after dragging Claude Owl to Applications:

xattr -r -d com.apple.quarantine "/Applications/Claude Owl.app"

This removes the download quarantine flag. After running this command, Claude Owl will launch normally.

üìù Why does this happen? Claude Owl is currently distributed without Apple code signing (which costs $99/year). macOS automatically quarantines downloaded unsigned apps and prevents them from launching. Code signing will be added in a future version.
